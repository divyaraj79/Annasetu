# ADR-000 — Project Foundation

**Status:** Accepted

**Date:** July 2026

---

# Purpose

This document establishes the overall vision, philosophy, and development principles of AnnaSetu.

It serves as the foundation for every architectural decision made throughout the project.

---

# What is AnnaSetu?

AnnaSetu is an AI-assisted food redistribution platform that connects restaurants with NGOs to reduce food wastage.

The platform is designed to simplify food donation while ensuring that available food reaches organizations capable of distributing it before expiry.

Unlike a traditional CRUD application, AnnaSetu is being built as a modular backend that will later be enhanced with AI-driven automation.

---

# Project Goals

The primary goals are:

- Reduce food wastage.
- Simplify restaurant food donation.
- Help NGOs receive food faster.
- Build a clean, maintainable backend.
- Keep AI separated from business logic.
- Produce a deployment-ready MVP.

---

# Development Philosophy

The project intentionally prioritizes:

- Clean architecture
- Proper database design
- Correct business rules
- Modular services
- Maintainability
- Future extensibility

instead of:

- Premature optimization
- Over-engineering
- Complex microservices
- AI-first implementation

---

# Core Principles

## 1. Backend First

The backend must be stable before introducing authentication, automation, or AI.

---

## 2. Separation of Concerns

Every layer has one responsibility.

Models define database structure.

Schemas validate API data.

Services contain business logic.

Routers expose API endpoints.

---

## 3. AI is an Enhancement

AI should orchestrate backend services.

AI should never replace backend business logic.

---

## 4. Incremental Development

Modules are completed one at a time.

Each module is reviewed before moving to the next.

---

## 5. Business Rules over CRUD

CRUD is only the starting point.

Every service will gradually evolve into domain-specific business logic.

---

# Current Development Stage

The project is currently focused on backend development.

Completed foundation work includes:

- Database Models
- Pydantic Schemas
- Layered Router → Service → Model architecture
- CRUD Foundation
- Authentication
- Authorization
- Validation
- Soft Delete
- Database Constraints
- Global Exception Handling
- Transaction Ownership

The backend foundation is now considered substantially complete.

Future development will focus on:

- Admin workflows
- AI Automation
- Geospatial services
- Notifications
- Frontend

Authentication, automation and AI are intentionally postponed until the backend foundation is complete.

# Decision

This philosophy will guide every future architectural decision.

Any future implementation should preserve:

- modularity
- maintainability
- scalability
- separation of concerns

rather than introducing shortcuts that increase coupling.
